# Developer Portfolio & Git Demonstration

A responsive, professional personal portfolio built using HTML, CSS, and vanilla JavaScript. 

This project was specifically developed to demonstrate advanced Git version control workflows, repository management, and safe history recovery techniques.

## ✨ Features
* **Modern UI:** Clean, off-white and off-black aesthetic with soft shadow components.
* **Responsive Layout:** CSS Grid and Flexbox used to adapt to different screen sizes.
* **Interactive Elements:** Smooth scrolling and JavaScript-handled form submission.
* **Complete Git History:** Contains a comprehensive commit history demonstrating feature branching, merge conflict resolution, and history recovery.

## 🚀 Setup & Installation
Because this project uses vanilla web technologies, no complex build tools or package managers are required.

1. **Clone the repository:**
   ```bash
   git clone [https://github.com/ahmed-sakil/my-portfolio.git](https://github.com/ahmed-sakil/my-portfolio.git)

2. **Navigate to the directory:**
   ```bash
   cd my-portfolio-git-project

3. **Run the project:**
Simply double-click the index.html file to open it in your default web browser, or use a tool like VS Code Live Server for hot-reloading.

##🛠️ Git Workflows Demonstrated
This repository serves as a practical assessment of the following Git operations:
* Feature Branching: Isolated development for HTML, CSS, and JS.
* Merge Conflicts: Intentional collision and resolution inside README.md.
* Safe Undos: Using git revert to safely undo changes without rewriting history.
* Disaster Recovery: Using git reflog and git cherry-pick to recover deleted commits from a hard reset.
* Release Tagging: Semantic versioning applied via Git tags (e.g., v1.0.0).
